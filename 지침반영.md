# Vercel 배포 문제 해결 및 지침 반영

본 문서는 `student-economy` 프로젝트 배포 과정에서 발생한 기술적 문제와 그 해결책을 정리하여, 향후 동일한 문제가 재발하지 않도록 지침을 제공합니다.

## 1. 주요 발생 문제 및 해결 방안

### ① CSS/PostCSS 설정 오류 (Autoprefixer)
- **문제**: `Module not found: Can't resolve 'autoprefixer'` 에러 발생 및 PostCSS 플러그인 로드 실패.
- **원인**: `package.json`에 `autoprefixer`가 명시되지 않았거나, `postcss.config.js`가 구버전 방식(Array형태)으로 작성됨.
- **해결**:
  - `npm install -D autoprefixer`로 명시적 설치.
  - `postcss.config.js`를 Object-based 형식으로 표준화.
```javascript
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

### ② 라이브러리 의존성 충돌 (Yahoo Finance 2)
- **문제**: `yahoo-finance2` 라이브러리가 Deno 전용 코드(@std/testing 등)를 포함하고 있어 Webpack 빌드 시 `Module not found` 발생.
- **원인**: Next.js(Node.js 환경)와 Deno 라이브러리 간의 호환성 부족.
- **해결**:
  - `next.config.mjs`의 `transpilePackages`에 추가.
  - Webpack Alias 설정을 통해 문제되는 모듈을 `false`로 처리(Nullify).
  - 최종적으로 빌드 안정을 위해 해당 API 호출 로직을 주석 처리(Bypass)하고 대체 방안 검토.

### ③ Middleware import 경로 오류
- **문제**: `Module not found: Can't resolve 'next/request'` 에러 발생.
- **원인**: Next.js 13/14 이후 `NextResponse`, `NextRequest`는 `next/server`에서 가져와야 함.
- **해결**: `import { NextResponse, type NextRequest } from 'next/server'`로 수정.

### ④ 의존성 버전 충돌 (React 19 vs Next 14)
- **문제**: `npm install` 시 Peer Dependency 에러로 설치 중단.
- **해결**: `.npmrc` 파일에 `legacy-peer-deps=true`를 설정하여 하위 호환성 강제 적용.

### ⑤ Git Lock 파일 이슈
- **문제**: 로컬 환경(Google Drive 등)에서 동기화 중 `.git/index.lock`이 남아 Git 명령어가 실행되지 않음.
- **해결**: `if (Test-Path .git\index.lock) { Remove-Item .git\index.lock -Force }` 명령어로 인덱스 락을 강제 제거 후 작업 진행.

## 2. 향후 프로젝트 반영 지침 (Best Practices)

1. **환경 변수 체크**: Supabase 등 외부 서비스 연동 시 서버/클라이언트 측 환경 변수가 모두 Vercel 대시보드에 설정되어 있는지 반드시 확인한다.
2. **패키지 호환성 검증**: Deno 기반 라이브러리나 서버 전용 패키지를 Next.js 클라이언트/Edge에서 사용할 때는 빌드 전 `transpilePackages` 설정을 미리 검토한다.
3. **빌드 로그 모니터링**: `npx vercel build --debug`를 사용하여 로컬에서 Vercel 빌드 환경을 최대한 시뮬레이션하고 상세 에러 로그를 확인한다.
4. **표준 Import 경로 준수**: Next.js 전용 타입과 유틸리티는 공식 문서의 최신 경로(예: `next/server`, `next/navigation`)를 사용한다.
